# ACSD-69016: El precio especial no se aplica en sitios web con diferentes zonas horarias

El parche ACSD-69016 corrige el problema en el que las solicitudes de API de actualización de producto [!DNL REST] fallan si no existe ningún almacén con el código predeterminado, lo que impide que los precios especiales surtan efecto para los sitios web creados en diferentes zonas horarias. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.77. El ID del parche es ACSD-69016. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8-p1

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7 - 2.4.8-p3

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Las solicitudes de API de actualización de producto [!DNL REST] fallan si no existe ningún almacén con el código predeterminado, lo que impide que los precios especiales surtan efecto para los sitios web creados en diferentes zonas horarias.

<u>Pasos a seguir</u>:

1. En el panel **[!UICONTROL Admin]**, vaya a **[!UICONTROL Stores]** >**[!UICONTROL &#x200B; All Stores]** > **[!UICONTROL Default Store View]**.
1. Cambie el valor de código de la vista de tienda predeterminada.
1. Enviar una solicitud de API [!DNL REST] para actualizar un producto.

<u>Resultados esperados</u>:

La solicitud se completa correctamente y el producto se actualiza.

<u>Resultados reales</u>:

La solicitud falla y el producto no se actualiza.
