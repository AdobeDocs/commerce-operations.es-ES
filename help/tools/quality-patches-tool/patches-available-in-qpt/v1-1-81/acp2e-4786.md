# ACP2E-4786: La exportación del producto falla cuando se configura el almacenamiento remoto de AWS S3

El parche ACP2E-4786 corrige el problema en el que la exportación del producto falla porque el controlador de almacenamiento utiliza una ruta no válida cuando el almacenamiento remoto de AWS S3 está habilitado. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.81. El ID del parche es ACP2E-4786. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7-p4

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.5 - 2.4.9

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

La exportación del producto falla cuando el almacenamiento remoto de AWS S3 está habilitado debido a la administración de rutas no válida en el controlador de almacenamiento.

<u>Pasos a seguir</u>:

1. Configurar [!DNL AWS S3] como almacenamiento remoto:

   ```
   bin/magento setup:config:set --remote-storage-driver="aws-s3" --remote-storage-bucket="..." --remote-storage-region="..." --remote-storage-key="..." --remote-storage-secret="..." -n
   ```

1. Deshabilite el almacenamiento de la base de datos para archivos multimedia ejecutando el siguiente comando:

   ```
   bin/magento config:set system/media_storage_configuration/media_database 0
   ```

1. En la barra lateral de administración, vaya a **[!UICONTROL System]** > **[!UICONTROL Data Transfer]** > **Exportar**.
1. Para **[!UICONTROL Entity Type]**, elija **[!UICONTROL Products]**.
1. Haga clic en **[!UICONTROL Continue]**.

<u>Resultados esperados</u>:

El archivo de exportación se ha creado correctamente.

<u>Resultados reales</u>:

1. No se ha creado el archivo de exportación.
1. Los registros muestran un error *SignatureDoesNotMatch*.
