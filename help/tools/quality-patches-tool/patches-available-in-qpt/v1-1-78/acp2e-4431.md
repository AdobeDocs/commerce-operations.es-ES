# ACP2E-4431: [!UICONTROL Related Products] coincidentes con reglas de destino se eliminan durante el proceso de reindexación

El parche ACP2E-4431 corrige el problema en el que **[!UICONTROL Related Products]** coincidentes con las reglas de destino se eliminan durante el proceso de reindexación. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. El ID del parche es ACP2E-4431. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7-p4

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Las **[!UICONTROL Related Products]** que coinciden con las reglas de destino se eliminan durante el proceso de reindexación.

<u>Pasos a seguir</u>:

1. Cree 10000 o más productos.
1. Asigne 10000 o más productos a 5000 categorías o más.
1. Cree 100 o más reglas de producto relacionadas.
1. Configure las condiciones de la regla donde **[!UICONTROL Products]** debe coincidir con el usuario **[!UICONTROL Product Category]** es *X*.
1. Configurar **[!UICONTROL Products]** para que se muestre con **[!UICONTROL Product Category]** es **[!UICONTROL Constant Value]** *Y*.
1. Ejecute esta consulta de GraphQL para recuperar `related_products` de un producto:

   ```graphql
   {
       products(filter: { url_key: { eq: "simple-product-491" } }) {
           items {
               sku
               related_products {
                   uid
                   sku
                   name
                   url_key
                   stock_status
                   price_range {
                       minimum_price {
                           regular_price {
                              value
                           }
                           final_price {
                               value
                           }
                       }
                   }
               }
           }
       }
   }
   ```

1. Cree un registro de pendientes grande para el índice `targetrule_product_rule`.
1. Cree un registro de pendientes grande para el índice `targetrule_rule_product`.
1. Ejecute el indizador `targetrule_rule_product`.
1. Ejecute el indizador `targetrule_product_rule`.

<u>Resultados esperados</u>:

La lista `related_products` siempre se rellena con productos que cumplen las condiciones de regla.

<u>Resultados reales</u>:

La lista `related_products` devuelve una matriz vacía mientras se ejecutan los indizadores.
