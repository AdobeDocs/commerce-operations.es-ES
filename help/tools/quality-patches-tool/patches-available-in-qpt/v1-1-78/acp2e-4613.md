# ACP2E-4613: El directorio de medios grande causa retrasos en la carga del árbol de directorios de Media Gallery

El parche ACP2E-4613 corrige el problema en el que las estructuras de directorios de medios grandes causan respuestas `gettree` lentas, lo que lleva a tiempos de carga prolongados para el árbol de directorios **[!UICONTROL Media Gallery]**. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. El ID del parche es ACP2E-4613. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8-p3

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Las estructuras de directorio de medios grandes causan respuestas `gettree` lentas, lo que lleva a tiempos de carga prolongados para el árbol de directorio [!UICONTROL Media Gallery].

<u>Requisitos previos</u>:

1. Existe un gran número de productos con varias imágenes.

<u>Pasos a seguir</u>:

1. Genere miniaturas de imagen para aumentar el espacio del medio:

   ```
   bin/magento catalog:images:resize
   ```

1. Cree varios subdirectorios en `pub/media/catalog/category` (por ejemplo: test, test2, test3, test4 y carpetas anidadas adicionales según sea necesario).
1. Copie un gran conjunto de archivos de medios de catálogo en los directorios recién creados para aumentar significativamente el tamaño del árbol de directorios de medios, por ejemplo:

   ```
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test/
   cp -r pub/media/catalog/product/* pub/media/catalog/category/test2/
   (...)
   ```

1. Inicie sesión en el panel de administración y vaya a **[!UICONTROL Content]** > *[!UICONTROL Media]* > **[!UICONTROL Media Gallery]**.
1. Observe que la solicitud `admin/media_gallery/directories/gettree/?isAjax=true` tarda mucho tiempo en completarse.

<u>Resultados esperados</u>:

El árbol de directorios se carga en unos segundos.

<u>Resultados reales</u>:

El árbol de directorios tarda varios minutos en cargarse, según el número de subdirectorios bajo `pub/media`.
