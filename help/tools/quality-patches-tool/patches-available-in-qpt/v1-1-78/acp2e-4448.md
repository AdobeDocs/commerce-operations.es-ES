# ACP2E-4448: Los cambios de configuración realizados durante las interrupciones de [!DNL Redis] no se reflejan después de que [!DNL Redis] se recupere y los valores antiguos persisten

El parche ACP2E-4448 corrige un problema en el que la caché local y la caché remota de Redis podrían dessincronizarse cuando Redis no está disponible, y los cambios de configuración realizados durante las interrupciones de Redis no se reflejan después de que Redis se recupera, lo que provoca que los valores antiguos persistan. La corrección garantiza la sincronización correcta en todas las cachés y conserva los valores guardados más recientes incluso cuando Redis está caído. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. El ID del parche es ACP2E-4448. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8-p1

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

La caché L2, la caché local, la caché obsoleta y Redis se desincronizaron cuando Redis no estaba disponible. Los cambios de configuración realizados durante las interrupciones de Redis no se reflejaron después de que Redis se recuperara, lo que provocó que los valores antiguos persistieran.

<u>Pasos a seguir</u>:

1. Configure la caché L2 usando el servidor `RemoteSynchronizedCache` con Redis como servidor remoto y la caché de archivos como servidor local.
1. Vaya a **Configuración** > **General** > **Contacto** > **Opciones de correo electrónico** > **Enviar correos electrónicos a**.
1. Actualice el valor **Enviar correos electrónicos a** a _100@example.com_.
1. Actualice la página varias veces para confirmar que el valor se almacena en caché.
1. Detenga el servicio Redis para simular una interrupción del servicio.
1. Actualizar **Enviar correos electrónicos a** valor a _200@example.com_.
1. Inicie el servicio Redis.
1. Actualice la página de configuración.

<u>Resultados esperados</u>:

Debe mostrar el nuevo valor de configuración _200@example.com_.

<u>Resultados reales</u>:

Muestra el valor de configuración antiguo _100@example.com_.
