# ACP2E-4609: La página **[!UICONTROL My Quotes]** no muestra comillas cuando algunas comillas contienen productos eliminados

El parche ACP2E-4609 corrige el problema en el que la página **[!UICONTROL My Quotes]** no muestra comillas cuando algunas comillas contienen productos eliminados. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. El ID del parche es ACP2E-4609. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7-p8

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.6 - 2.4.8-p4

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

La página **[!UICONTROL My Quotes]** no muestra comillas cuando algunas comillas contienen productos eliminados.

<u>Requisitos previos</u>:

Las funciones de empresa y presupuesto B2B están activadas.

<u>Pasos a seguir</u>:

1. Cree una empresa.
1. Cree varios productos simples.
1. Cree un producto agrupado con al menos dos opciones y seleccione **[!UICONTROL User Defined]**. Asigne los productos simples a diferentes opciones de paquete.
1. Inicie sesión en la tienda como administrador de la empresa.
1. Añada el producto del paquete al carro de compras dos veces utilizando las mismas opciones pero con cantidades diferentes.
1. Vaya al carro de compras y cree un presupuesto.
1. Eliminar los productos simples asociados.
1. Para reproducir el problema, la cotización debe marcarse como `trigger_recollect: 1`.

   Ejecute el siguiente comando SQL en la base de datos de Adobe Commerce:

   ```
   UPDATE quote SET trigger_recollect = 1 WHERE entity_id = 1;
   ```

1. Ir a **[!UICONTROL My Account]** > **[!UICONTROL My Quotes]**.

<u>Resultados esperados</u>:

La cuadrícula **[!UICONTROL My Quotes]** se debe representar correctamente.

<u>Resultados reales</u>:

1. La cuadrícula **[!UICONTROL My Quotes]** no se ha rellenado.
1. Respuesta HTTP (`negotiable_quote/ui/render/`):

   ```
   {"error":"Requested quote is not found. Row ID: quoteId = 1","errorcode":"0"}
   ```

1. Error en `exception.log`:

   ```
   report.CRITICAL: Magento\Framework\Exception\NoSuchEntityException: Requested quote is not found. Row ID: quoteId = 1 in app/code/Magento/NegotiableQuote/Model/NegotiableQuoteManagement.php:673
   ```
