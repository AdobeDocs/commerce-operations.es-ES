# ACP2E-4507: La configuración de [!UICONTROL Password Options] no se aplica a las solicitudes de restablecimiento de contraseña de cliente realizadas a través de mutaciones de GraphQL

El parche ACP2E-4507 corrige el problema en el que la configuración **[!UICONTROL Password Options]** no se aplica a las solicitudes de restablecimiento de contraseña de cliente realizadas a través de mutaciones de GraphQL. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. El ID del parche es ACP2E-4507. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8-p3

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8 - 2.4.8-p4

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

La configuración **[!UICONTROL Password Options]** no se aplica a las solicitudes de restablecimiento de contraseña de cliente realizadas a través de mutaciones de GraphQL.

<u>Pasos a seguir</u>:

1. Compruebe y confirme las opciones de contraseña de cliente en
   **[!UICONTROL Store]** > **[!UICONTROL Configurations]** > **[!UICONTROL Customers]** > **[!UICONTROL Customer Configuration]** > **[!UICONTROL Password Options]**.
1. Envíe un correo electrónico para restablecer su contraseña utilizando la mutación de GraphQL `requestPasswordResetEmail`.
1. Restablezca la contraseña utilizando la mutación de GraphQL `resetPassword` con el token del correo electrónico enviado.
1. Repita los mismos pasos para restablecer la contraseña varias veces.

<u>Resultados esperados</u>:

Los restablecimientos de contraseña de cliente activados mediante GraphQL se comportan del mismo modo que los restablecimientos de contraseña basados en web y respetan los ajustes de configuración de [!UICONTROL Admin].

<u>Resultados reales</u>:

Las solicitudes de restablecimiento de contraseña realizadas a través de GraphQL se pueden activar un número ilimitado de veces y no aplican los límites definidos en la configuración de contraseña del cliente de [!UICONTROL Admin].
