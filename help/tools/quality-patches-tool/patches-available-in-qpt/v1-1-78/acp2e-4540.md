# ACP2E-4540: La biblioteca Fotorama no se carga correctamente, por lo que solo es visible la primera imagen adjunta

El parche ACP2E-4540 corrige el problema en el que la biblioteca Fotorama no se carga correctamente, lo que provoca que solo se muestre la primera imagen adjunta. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.78. El ID del parche es ACP2E-4540. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7-p7

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7 - 2.4.8-p4

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

La biblioteca Fotorama no se carga correctamente, lo que hace que solo sea visible la primera imagen adjunta.

<u>Pasos a seguir</u>:

1. Cree un producto sencillo y adjúntele varias imágenes.
1. Abra el producto en la tienda.

<u>Resultados esperados</u>:

La galería de imágenes muestra todas las imágenes adjuntas.

<u>Resultados reales</u>:

1. Solo se ve la primera imagen.
1. La consola del explorador muestra el siguiente error:

```
Javascript error(severe) - http://magento.com/static/version1717863530/frontend/magento/luma/en_us/mage/gallery/gallery.js 309:29 uncaught type error: $fotorama element.fotorama is not a function
```
