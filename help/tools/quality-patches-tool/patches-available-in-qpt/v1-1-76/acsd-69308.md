# ACSD-69308: Las reglas de precios de catálogo no se aplican cuando `special_price` se establece en el nivel de sitio web

La revisión ACSD-69308 corrige el problema en el cual las reglas de precios de catálogo no se aplicaban cuando `special_price` se establecía solamente en el nivel de sitio web (no en &quot;Todas las vistas de la tienda&quot;). Después de la corrección, las reglas de precios de catálogo se aplican correctamente comprobando primero la tienda predeterminada del sitio web. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76. El ID del parche es ACSD-69308. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Las reglas de precios de catálogo no se aplicaban cuando `special_price` se configuraba solamente en el nivel de sitio web. El sistema no reconocía correctamente el precio de los productos asignados a una vista de tienda específica, lo que daba como resultado precios incoherentes.

<u>Pasos a seguir</u>:

1. Inicie sesión en el panel de administración.
1. Crear un sitio web, una tienda y una vista de tienda personalizados (web2).
1. Establezca **[!UICONTROL Catalog Price Scope]** en *Sitio web*.
1. Establezca **[!UICONTROL Use for Promo Rule Conditions]** en *Yes* para el atributo `special_price`.
1. Cree la siguiente regla de precios de catálogo.

```
Website: web2
Conditions:
Special Price  greater than  15
Special Price  less than  50
Actions:
Apply: Apply as percentage of original
Discount Amount: 100
```

1. Cree dos productos asignados solamente a la vista de tienda **web2**.

Producto 01:

```
Set Advanced Pricing:
All Store View Scope: 30.00
web2 scope: 40.00
```

Producto 02:

```
Set Advanced Pricing:
All Store View Scope: keep empty
web2 scope: 40.00
```

1. Reindexe y cargue cada producto en la tienda en la vista de la tienda **web2**.

<u>Resultados esperados</u>:

La regla de precio de catálogo debe aplicarse a la vista de tienda web2. El producto 02 debería mostrar un precio de *0.00*, porque su precio especial coincide con la condición de regla del catálogo.

<u>Resultados reales</u>:

* Producto 01: El precio se muestra como *0.00*, que coincide con la condición de regla del catálogo.
* Producto 02: El precio se muestra como *40.00*, que es **inesperado**, ya que la regla de catálogo debe aplicarse en función del precio especial establecido en el nivel de sitio web.
