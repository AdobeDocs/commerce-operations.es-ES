# ACSD-69129: la actualización de precios del nivel de API [!DNL REST] falla después de eliminar el sitio web base predeterminado y de usar el sitio web secundario como predeterminado

El parche ACSD-69129 corrige el problema que se producía al eliminar el sitio web base predeterminado y al usar el sitio web secundario como predeterminado, se producía un error al intentar actualizar el precio de nivel del sitio web secundario mediante la API [!DNL REST]. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.76. El ID del parche es ACSD-69129. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.6-p13, 2.4.8-p3

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.5 - 2.4.6-p13, 2.4.8 - 2.4.8-p3

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Si se elimina el sitio web base predeterminado y se utiliza el sitio web secundario como predeterminado, se produce un error al intentar actualizar el precio de nivel del sitio web secundario mediante la API [!DNL REST].

<u>Pasos a seguir</u>:

1. Inicie sesión en el panel [!UICONTROL Admin].
1. Conjunto **[!UICONTROL Catalog Price Scope]** = *Sitio web*.
1. Cree un nuevo sitio web, tienda o vista de tienda.
1. Establezca *Establecer como predeterminado* para el sitio web recién creado.
1. Borre la caché.
1. Cree un nuevo producto. Asígnelo únicamente al segundo sitio web.
1. Envíe una solicitud de API [!DNL REST] para establecer el precio de nivel del segundo sitio web.
1. El resultado debe ser una matriz vacía con una respuesta de *200*.
1. En [!UICONTROL Admin], elimine *Sitio web principal*.
1. Envíe una solicitud de API [!DNL REST] para establecer el precio de nivel del segundo sitio web.

<u>Resultados esperados</u>:

* Se recibió una respuesta de *200*.
* Se actualiza el precio de nivel del producto.

<u>Resultados reales</u>:

* El atributo *Id. de sitio web no válido = %websiteId. Identificador de fila: SKU = %SKU, ID de sitio web: %websiteId, grupo de clientes: %customerGroup, cantidad: %qty.Se devuelve el error* en la respuesta.
* El precio de nivel del producto no se ha actualizado.
