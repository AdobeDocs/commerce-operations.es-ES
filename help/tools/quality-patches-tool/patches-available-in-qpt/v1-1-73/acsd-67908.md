# ACSD-67908: Los archivos de JavaScript no se combinan correctamente en configuraciones de varias tiendas

El parche ACSD-67908 soluciona el problema de que los archivos JavaScript no se combinan correctamente en configuraciones de varias tiendas. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.73. El ID del parche es ACSD-67908. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7-p5

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.4 - 2.4.8-p3

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Los archivos JavaScript combinados no se generan correctamente en configuraciones de varias tiendas, lo que provoca archivos dañados y errores de tienda.

<u>Pasos a seguir</u>:

1. Configurar 50 o más sitios web, tiendas y vistas de tiendas.
1. Establecer modo de producción y omitir compilación:

   ```
   bin/magento deploy:mode:set production --skip-compilation
   ```

1. Implementar código generado:

   ```
   bin/magento setup:di:compile
   ```

1. Implemente contenido estático mediante una estrategia compacta:

   ```
   bin/magento setup:static-content:deploy --strategy compact --jobs 10
   ```

1. Eliminar contenido de la carpeta de caché combinada:

   ```
   rm -rf pub/static/_cache/merged/*
   ```

1. Borrar caché:

   ```
   bin/magento cache:clean
   ```

1. Abra todas las vistas de la tienda simultáneamente en fichas o ventanas de explorador independientes.


<u>Resultados esperados</u>:

Todos los archivos de JavaScript se combinan correctamente y la tienda funciona sin errores.

<u>Resultados reales</u>:

Algunos archivos JavaScript combinados están dañados y no comienzan con la declaración del módulo RequireJS correcta.
