# ACP2E-4472: Presupuesto nulo creado mediante el flujo Iniciar sesión como cliente

El parche ACP2E-4472 corrige el problema en el que se crean registros de comillas nulas en la tabla de comillas durante el flujo **[!UICONTROL Login as Customer]**. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80. El ID del parche es ACP2E-4472. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7-p8

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Se crea un registro de comillas nulas en la tabla de comillas durante el flujo [!UICONTROL Login as Customer].

<u>Pasos a seguir</u>:

1. Cree dos clientes.
1. Vaya a **[!UICONTROL Admin]** > **[!UICONTROL Customers]** > **[!UICONTROL All Customers]**.
1. En la cuadrícula de clientes, seleccione un cliente y haga clic en **[!UICONTROL Login as Customer]**.
1. Después de ser redirigido a la tienda, compruebe los datos en la tabla de la base de datos `quote`.
1. Sin cerrar la sesión de tienda actual, vuelva a la cuadrícula de clientes **[!UICONTROL Admin]**, seleccione el otro cliente y haga clic en **[!UICONTROL Login as Customer]**.

<u>Resultados esperados</u>:

En la tabla de la base de datos `quote` solo se crea un presupuesto asociado con el cliente que ha iniciado sesión.

<u>Resultados reales</u>:

Junto con la oferta del cliente, se crea un registro de oferta nulo en la tabla de base de datos `quote`.
