# ACP2E-4533: Las imágenes de marcador de posición no se pueden cargar en la tienda cuando la URL incluye el código de tienda

El parche ACP2E-4533 corrige el problema en el que las imágenes de marcador de posición no se cargan en la tienda cuando se incluye un código de tienda en la URL. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80. El ID del parche es ACP2E-4533. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7-p8

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7 - 2.4.7-p10

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Las imágenes de marcador de posición no se cargan en la tienda cuando la dirección URL incluye el código de tienda.


<u>Pasos a seguir</u>:

1. Cree un sitio web, una tienda y una vista de tienda adicionales.
1. Para configurar imágenes de marcador de posición en el ámbito predeterminado, en la barra lateral de administración, vaya a **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]**.
1. En el panel izquierdo, expanda **[!UICONTROL Catalog]** y elija **[!UICONTROL Catalog]**.
1. Expanda la sección **[!UICONTROL Product Image Placeholders]** para configurar las imágenes de marcador de posición para el nuevo ámbito del sitio web.
1. Cree un producto con imágenes y asígnelo a ambos sitios web.
1. Eliminar las imágenes de producto cargadas de la carpeta `pub/media/catalog/product`.
1. Abra el producto en la tienda para ambos sitios web.

<u>Resultados esperados</u>:

El sitio web no predeterminado debe mostrar las imágenes de marcador de posición específicas del sitio web cuando las imágenes reales no estén disponibles.

<u>Resultados reales</u>:

El sitio web no predeterminado muestra la imagen de marcador de posición predeterminada, independientemente de las imágenes de marcador de posición cargadas para él.
