# ACP2E-4610: Ejecución lenta de `sales_clean_quotes` cron

El parche ACP2E-4610 corrige el problema en el que el trabajo cron de `sales_clean_quotes` tiene problemas de rendimiento. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80. El ID del parche es ACP2E-4610. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.6-p13

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.5 - 2.4.8-p5

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

El trabajo cron `sales_clean_quotes` tiene problemas de rendimiento.

<u>Pasos a seguir</u>:

1. Genere varias ofertas caducadas en la tabla de ofertas.
1. Ejecute el trabajo cron `sales_clean_quotes` y mida el tiempo de ejecución.

<u>Resultados esperados</u>:

El trabajo cron de `sales_clean_quotes` finaliza en un tiempo razonable.

<u>Resultados reales</u>:

El trabajo cron de `sales_clean_quotes` tarda mucho tiempo en finalizar.
