# ACP2E-4156: La validación de la dirección de envío de la API [!DNL REST] ignora la configuración del atributo de administración

El parche ACP2E-4156 corrige el problema en el que la validación de la dirección de envío en la API [!DNL REST] no se adhiere a la configuración de atributo definida en Administración. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80. El ID del parche es ACP2E-4156. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8-p4

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8 - 2.4.8-p5

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

En la API [!DNL REST], la validación de direcciones de envío no respeta la configuración de atributos de direcciones de clientes establecida en Administración.

<u>Pasos a seguir</u>:

1. Vaya a **[!UICONTROL Stores]** > **[!UICONTROL Attributes]** > **[!UICONTROL Customer Address]** en el panel de administración.
1. Abra el atributo con el código de atributo *firstname*.
1. En **[!UICONTROL Attribute Properties]**, establezca:
   1. **[!UICONTROL Input Validation]**: Solo Longitud.
   1. **[!UICONTROL Minimum Text Length]**: *1*.
   1. **[!UICONTROL Maximum Text Length]**: *20*.
1. Vaya a la tienda y añada el producto de prueba al carro de compras.
1. Envíe una solicitud de API [!DNL REST] a `/rest/default/V1/guest-carts/cartId/shipping-information` para agregar o actualizar la información de envío con un valor de *firstname* superior a 20 caracteres.

<u>Resultados esperados</u>:

Se devuelve un error que indica que el valor *firstname* supera el límite de longitud configurado.

<u>Resultados reales</u>:

La información de envío del carro de compras se actualizó correctamente con el valor *firstname* enviado en la solicitud de API [!DNL REST].
