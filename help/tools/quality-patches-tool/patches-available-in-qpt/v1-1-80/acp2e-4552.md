# ACP2E-4552: La respuesta de GraphQL no devuelve el estado de la empresa

El parche ACP2E-4552 corrige el problema en el que el estado de la empresa no se devuelve en la respuesta de GraphQL. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.80. El ID del parche es ACP2E-4552. Tenga en cuenta que este problema está programado para solucionarse en Adobe Commerce B2B 1.5.2-p3.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) B2B 1.5.2-p3

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) B2B >=1.5.0 &lt;1.5.3

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

La respuesta de GraphQL no devuelve el estado de la empresa.

<u>Requisitos previos</u>:

[!DNL B2B] está instalado.

<u>Pasos a seguir</u>:

1. En el Administrador, vaya a **[!UICONTROL Stores]** > *[!UICONTROL Settings]* > **[!UICONTROL Configuration]** > **[!UICONTROL General]** > **[!UICONTROL B2B Features]** y establezca **[!UICONTROL Enable Company]** en `Yes`.
1. Inicie sesión en el administrador de Commerce como administrador.
1. Cree tres empresas con estos estados: Activo, Rechazado y Pendiente de aprobación.
1. En el Administrador, vaya a **[!UICONTROL Customers]** > **[!UICONTROL All Customers]**. Abra una cuenta de cliente y, en la lista **[!UICONTROL Associate to Company]**, asigne el cliente a cada una de las empresas creadas.
1. Envíe la siguiente solicitud de GraphQL:

   ```
   {
     customer {
       firstname
       lastname
       email
       status
       companies {
        items {
           id
           legal_name
           name
           status
        }
       }
     }
   }
   ```

<u>Resultados esperados</u>:

La respuesta de GraphQL devuelve los valores de estado correctos (como Activo, Pendiente de aprobación, Rechazado y Bloqueado) para todas las empresas asociadas con el cliente. La respuesta también se puede filtrar u ordenar por estado de la compañía.

<u>Resultados reales</u>:

La respuesta de GraphQL no incluye los valores de estado de la empresa para las empresas asociadas con el cliente.
