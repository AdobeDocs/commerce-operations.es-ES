# ACSD-68793: los productos válidos se rechazan incorrectamente durante la asignación del catálogo compartido

El parche ACSD-68793 corrige el problema en el que los productos válidos se rechazan incorrectamente durante la asignación del catálogo compartido. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.75. El ID del parche es ACSD-68793. Tenga en cuenta que este problema está programado para solucionarse en Adobe Commerce [!DNL B2B] 1.5.3.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) [!DNL B2B] 1.5.2-p1

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) [!DNL B2B] 1.5.1 - 1.5.2-p3

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches ](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Los productos válidos se rechazan incorrectamente durante la asignación del catálogo compartido mediante la API [!DNL REST] y el sistema devuelve un error que indica que no existen SKU existentes.

<u>Pasos a seguir</u>:

1. Cree varios productos.
1. En la barra lateral **[!UICONTROL Admin]**, vaya a **[!UICONTROL Catalog]** > **[!UICONTROL Shared Catalog]**.
1. Haga clic en **[!UICONTROL Add Shared Catalog]** y cree un catálogo compartido.
1. Asigne todos los productos al catálogo compartido que ha creado.
1. Envíe una solicitud de API [!DNL REST] a `/rest/V1/sharedCatalog/3/assignProducts` con la lista de SKU de productos existentes para asignarlos al catálogo compartido.

<u>Resultados esperados</u>:

Se devuelve una respuesta correcta. Los errores solo deben devolverse en el caso de los productos que no existen.

<u>Resultados reales</u>:

El sistema devuelve el error *El producto solicitado no existe: %sku.* para un SKU existente.
