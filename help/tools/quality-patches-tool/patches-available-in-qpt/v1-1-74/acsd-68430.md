# ACSD-68430: Error al guardar un cliente o una dirección de cliente si el registro incluía varias opciones de atributo que se habían eliminado de la configuración de atributos

El parche ACSD-68430 corrige un problema en el que al guardar una dirección de cliente o de cliente se produce un error si el registro incluye varias opciones de atributos que se eliminaron de la configuración de atributos. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.74. El ID del parche es ACSD-68430. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.7-p6

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.6 - 2.4.7-p8

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Al guardar un cliente o una dirección de cliente que incluye varias opciones de atributos que se eliminaron de la configuración de atributos, la operación falla.

<u>Pasos a seguir</u>:

1. Vaya a Administración y cree un atributo de cliente personalizado multiselección (por ejemplo: &quot;prueba&quot;) con las opciones A, B, C y D.
1. Cree un cliente desde Administración y guarde los valores de este atributo (por ejemplo: seleccione A, B, C).
1. Vuelva a la configuración de atributos y elimine una de las opciones que se guardaron para el cliente (por ejemplo: elimine la opción C).
1. En Administración, intente editar el registro del mismo cliente. Cambie cualquier campo (como sexo o prefijo) y haga clic en **[!UICONTROL Save]**.

<u>Resultados esperados</u>:

Los cambios del registro de cliente se guardan sin errores.

<u>Resultados reales</u>:

Se produce un error al guardar los cambios en el registro del cliente: *El atributo [attribute_name] no contiene la opción con el identificador [deleted_option_id]*.
