# ACP2E-4531: La jerarquía de CMS no actualiza la URL jerárquica cuando cambia la clave de URL de la página secundaria

El parche ACP2E-4531 soluciona el problema de que al cambiar la clave URL de una página CMS no se actualizaba la URL jerárquica de la página. Esta revisión está disponible cuando está instalado [[!DNL Quality Patches Tool (QPT)]](/help/tools/quality-patches-tool/quality-patches-tool-to-self-serve-quality-patches.md) 1.1.79. El ID del parche es ACP2E-4531. Este problema está programado para solucionarse en Adobe Commerce 2.4.9.

## Productos y versiones afectados

**El parche se ha creado para la versión de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.8-p3

**Compatible con versiones de Adobe Commerce:**

* Adobe Commerce (todos los métodos de implementación) 2.4.4 - 2.4.8-p4

>[!NOTE]
>
>El parche podría ser aplicable a otras versiones con las nuevas versiones de [!DNL Quality Patches Tool]. Para comprobar si el parche es compatible con su versión de Adobe Commerce, actualice el paquete `magento/quality-patches` a la última versión y compruebe la compatibilidad en la página [[!DNL Quality Patches Tool]: buscar parches &#x200B;](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html?lang=es). Utilice el ID de parche como palabra clave de búsqueda para localizar el parche.

## Problema

Al cambiar la clave URL de una página CMS no se actualiza la dirección URL jerárquica de la página.

<u>Pasos a seguir</u>:

1. Cree una página de CMS (por ejemplo, *ayuda*).
1. Cree una segunda página de CMS (por ejemplo, *my-account*) y establezca *help* como su elemento principal en la jerarquía.
1. Cargue ambas páginas de la Tienda y ambas se resolverán correctamente.
1. Cambie la clave URL de la página my-account (en *Optimización del motor de búsqueda*) a *my-account-2*.
1. Cargar `/my-account-2` y `/help/my-account-2` de la tienda.

<u>Resultados esperados</u>:

Cuando la clave de URL de la página CMS cambia, las URL de jerarquía se actualizan en consecuencia. `/help/my-account-2` se resuelve en la página y `/my-account` ya no se resuelve (no existe dicha dirección URL).

<u>Resultados reales</u>:

`/my-account-2` redirige a `/my-account` y `/help/my-account-2` devuelve un error 404. La dirección URL jerárquica no se actualiza cuando cambia la clave URL de la página secundaria.
